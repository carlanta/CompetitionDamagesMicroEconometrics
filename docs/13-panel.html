13  Datos de panel y efectos fijos

13.1 Seguir a muchas unidades durante mucho tiempo

Los datos de panel —observaciones repetidas de múltiples unidades a lo largo del tiempo— son el formato natural de muchos procedimientos de daños antitrust. En un cártel que afecta a varios mercados geográficos, el perito dispone de precios mensuales para cada territorio. En un abuso de posición dominante que perjudica a varias empresas, se observan márgenes trimestrales para cada víctima. En una práctica de exclusión con alcance parcial, algunos clientes están afectados y otros no, y ambos grupos se observan durante varios años.

Los modelos de efectos fijos permiten controlar la heterogeneidad inobservable entre unidades —diferencias permanentes en costes, calidad, localización, gestión— que podrían confundir la estimación del efecto de la conducta. Al “descontar” esas diferencias fijas, el modelo identifica el efecto de la conducta a partir de la variación temporal dentro de cada unidad.

13.2 La intuición: cada empresa es su propio control

La idea es simple: si cada unidad tiene un nivel estructural propio (una empresa siempre tiene costes más altos que otra, un mercado siempre tiene precios más bajos que otro), los efectos fijos absorben esas diferencias y el modelo se concentra en lo que cambia dentro de cada unidad a lo largo del tiempo.

Code 
set.seed(1313)

n_mercados <- 8
n_periodos <- 48
tratados <- c("M1", "M2", "M3", "M4")

df_panel <- expand_grid(
  mercado = paste0("M", 1:n_mercados),
  periodo = 1:n_periodos
) |>
  mutate(
    fecha = as.Date("2019-01-01") %m+% months(periodo - 1),
    mercado_id = match(mercado, unique(mercado)),
    fe_mercado = c(-8, -3, 0, 4, -5, 2, 6, -2)[mercado_id],
    tratado = if_else(mercado %in% tratados, 1, 0),
    post = if_else(periodo > 24, 1, 0),
    tendencia = 0.4 * periodo,
    estacionalidad = 3 * sin(2 * pi * periodo / 12),
    efecto = if_else(tratado == 1 & post == 1, -10 - 0.15 * (periodo - 24), 0),
    precio = 50 + fe_mercado + tendencia + estacionalidad + efecto + rnorm(n(), 0, 2)
  )

df_plot <- df_panel |>
  group_by(fecha, grupo = if_else(tratado == 1, "Tratados", "Control")) |>
  summarise(precio = mean(precio), .groups = "drop")

ggplot(df_plot, aes(x = fecha, y = precio, color = grupo)) +
  annotate("rect", xmin = as.Date("2021-01-01"), xmax = max(df_plot$fecha),
           ymin = -Inf, ymax = Inf, fill = "#e74c3c", alpha = 0.06) +
  geom_line(linewidth = 0.95) +
  geom_vline(xintercept = as.Date("2021-01-01"), color = "grey50", linetype = "dashed") +
  scale_color_manual(values = c("Tratados" = "#e74c3c", "Control" = "#2c3e50")) +
  labs(x = NULL, y = "Precio medio", color = NULL) +
  theme_minimal(base_size = 13) +
  theme(legend.position = "bottom", panel.grid.minor = element_blank())

13.3 La formulación econométrica

\[ Y_{it} = \alpha_i + \lambda_t + \delta D_{it} + \gamma X_{it} + \varepsilon_{it} \]

donde \(\alpha_i\) son efectos fijos de unidad, \(\lambda_t\) efectos fijos de tiempo, \(D_{it}\) es la variable de tratamiento y \(X_{it}\) son controles adicionales. El parámetro \(\delta\) es el efecto de la conducta.

Code 
# Pooled OLS
fit_ols <- lm(precio ~ tratado * post, data = df_panel)

# Efectos fijos de mercado + tiempo (TWFE)
fit_fe <- feols(precio ~ tratado:post | mercado + periodo, data = df_panel)

# Con errores clusterizados
se_cluster <- sqrt(vcov(fit_fe, cluster = "mercado")["tratado:post", "tratado:post"])

tibble(
  Modelo = c("OLS agrupado", "TWFE (fixest)", "TWFE + cluster"),
  Estimación = c(round(coef(fit_ols)["tratado:post"], 2),
                  round(coef(fit_fe)["tratado:post"], 2),
                  round(coef(fit_fe)["tratado:post"], 2)),
  `Error est.` = c(round(summary(fit_ols)$coefficients["tratado:post", "Std. Error"], 2),
                    round(sqrt(vcov(fit_fe)["tratado:post", "tratado:post"]), 2),
                    round(se_cluster, 2))
) |>
  knitr::kable(format = "pipe", align = c("l", "r", "r"))

13.4 Implementación en R

Code 
library(fixest)

# TWFE con efectos fijos y errores clusterizados
fit <- feols(precio ~ tratamiento | empresa + periodo,
             cluster = ~ empresa, data = panel)

# Test de Hausman (plm)
library(plm)
pdata <- pdata.frame(panel, index = c("empresa", "periodo"))
fit_fe_plm <- plm(precio ~ tratamiento, data = pdata, model = "within")
fit_re_plm <- plm(precio ~ tratamiento, data = pdata, model = "random")
phtest(fit_fe_plm, fit_re_plm)

13.5 Relevancia jurídica

Los modelos de panel son la base técnica de los métodos DID y comparables en casos multimercado. En procedimientos de cárteles que afectan a múltiples territorios o productos, el panel permite explotar la variación cruzada y temporal simultáneamente. La Guía Práctica de la Comisión Europea recomienda el uso de datos de panel cuando estén disponibles, por su capacidad para controlar la heterogeneidad inobservable.

13.7 Errores frecuentes

El error más común es estimar un OLS agrupado cuando existen diferencias permanentes entre unidades: el coeficiente queda sesgado por la heterogeneidad omitida. El segundo error es no clusterizar los errores estándar, lo que puede generar una falsa sensación de precisión. El tercero es aplicar TWFE con tratamientos escalonados sin comprobar la robustez frente a efectos heterogéneos en el tiempo.

13.8 Lo que un tribunal debe retener

Los datos de panel permiten seguir a muchas unidades durante mucho tiempo y separar el efecto de la conducta de las diferencias permanentes entre ellas. Cuando se combinan con efectos fijos de unidad y de tiempo, el modelo absorbe la heterogeneidad inobservable y ofrece una estimación del daño que es robusta a muchos de los factores de confusión que debilitarían otros enfoques.
